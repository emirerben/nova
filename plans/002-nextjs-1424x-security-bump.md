## Why this matters

The web app exact-pins `"next": "14.2.3"` — roughly two years of 14.2.x patch releases behind, which includes the fix window for CVE-2025-29927 (the middleware-authorization-bypass class) plus accumulated cache-poisoning, DoS, and image-optimizer fixes. This matters more than usual here because **Nova's admin authentication is implemented in Next.js middleware**: `src/apps/web/src/middleware.ts` enforces basic auth on `/admin/*` and `/api/admin/*`, and CLAUDE.md documents it as the mandatory, fail-closed admin gate. The app deploys on Vercel, which mitigates CVE-2025-29927 at the edge — so this is not an active emergency — but staying on an unpatched-for-two-years exact pin of the exact component that guards admin is cheap-to-remove risk. A within-minor bump (14.2.3 → latest 14.2.x) is a patch-range upgrade with no expected API changes.

### Step 3: Run the full local gate

```
cd src/apps/web && npm run lint && npx tsc --noEmit && npm test && npm run build
```

**Verify**: every command exits 0. `npm run build` completing matters most — it exercises the framework end-to-end including the middleware compilation.

### Step 4: Verify the admin middleware gate still fail-closes

Check whether a middleware test exists: `ls src/apps/web/src/__tests__ | grep -i middleware` (also check colocated `*.test.ts` next to `middleware.ts`). If one exists, `npm test -- middleware` → passes. If none exists, do a manual check: start the dev server (`npm run dev` from `src/apps/web` with `ADMIN_BASIC_AUTH_USER`/`ADMIN_BASIC_AUTH_PASSWORD` set in the env), then:

```
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/admin/templates
```

**Verify**: `401` (or `503` if the basic-auth env vars are unset — both prove the gate is active; `200` is a FAILURE). Stop the dev server afterwards.
